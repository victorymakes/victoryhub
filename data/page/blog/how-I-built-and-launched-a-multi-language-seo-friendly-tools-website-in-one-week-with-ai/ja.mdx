## ⚙️ スタック：スピードとSEOのために構築

- **フレームワーク：** Next.js 15（App Router）
- **ホスティング：** Cloudflare Pages
- **CSS：** Tailwind
- **UI：** Shadcn
- **AI：** GitHub Copilot + Claude
- **CI/CD：** GitHub → Cloudflare アダプター（`opennextjs-cloudflare`）

なぜこの組み合わせか？

- App Router = 組み込みのレイアウト、メタデータ、ロケールによるルーティング
- Cloudflare = エッジデプロイ、ゼロコスト、高速

バックエンドなし、DBなし、サーバーコストなし。純粋なユーティリティ提供のみ。

## ✨ AIが10倍の助けになった：その方法

> "AIはあなたを置き換えるのではなく、退屈な部分を置き換えるのです。"

VS Code内で**GitHub Copilot**を使用して自動補完しました：

- フォーム入力と状態ハンドラー
- ユーティリティ関数（UUID、エンコーダーなど）
- ボイラープレートページコンポーネント
- ブログ投稿テンプレート

そして**Claude**（エージェントスタイル）を使用しました：

- "ブラウザでクライアントサイドで画像を圧縮するにはどうすればいいですか？"
- "cronの文字列を読みやすいテキストに解析する最良の方法は何ですか？"
- "このi18nファイルを日本語と韓国語に翻訳してもらえますか？"

Claudeには以下も書いてもらいました：

- FAQ
- ブログ投稿の導入部
- 15言語の`meta`タグテンプレート
- SEO用のJSON-LDスキーマ

Copilot = コード生成
Claude = プロンプトベースのアシスタント
一緒に？ソロ開発者のチートコードです。

## 🌍 多言語：AI + 手動レビュー

> "翻訳はAIで簡単です。質の高い翻訳にはまだ努力が必要です。"

VictoryHubは**15言語**をサポートしています：
🇬🇧 🇨🇳 🇯🇵 🇰🇷 🇩🇪 🇫🇷 🇪🇸 🇷🇺 🇳🇱 🇮🇹 🇵🇹 🇳🇴 🇸🇪 🇫🇮 🇩🇰

私のワークフロー：

1. すべてのテキストを`en.json`に
2. Claudeを使用して各ターゲットロケールに翻訳
3. コンテキスト依存の単語を手動でレビュー
4. スクリプトを介して不足しているキーを自動チェック

App Routerの`i18n`ルーティング + 動的インポートでローカライズされたパスがクリーンになりました。

SEOボーナス：各ルートに`hreflang` + `alternate`タグを追加しました。

## 🔍 初日からのSEO：コンテンツの前に構造

ツールサイトを構築していて**SEOについて考えていない**なら、その可能性を無駄にしています。

早期に公開したもの：

- ✅ `next-sitemap`による`sitemap.xml`
- ✅ `robots.txt`
- ✅ ツール + FAQのJSON-LD構造化データ
- ✅ 15言語でのページごとのメタタイトル/説明（AI作成）
- ✅ 正規 + `hreflang`タグ

> "SEOは遅いと思いますか？公開*後*にやってみてください。"

### 価値のあるヒント：

AI用のプロンプト：

> "\[言語\]で155文字以内の\[ツール\]ページのSEOメタ説明を書いてください。"

驚くほどうまくいきます。まだ調整していますが、10倍速いです。

## ⚙️ Stack: Construido para velocidad y SEO

- **Framework:** Next.js 15 (App Router)
- **Hosting:** Cloudflare Pages
- **CSS:** Tailwind
- **UI:** Shadcn
- **IA:** GitHub Copilot + Claude
- **CI/CD:** GitHub → Adaptador Cloudflare (`opennextjs-cloudflare`)

¿Por qué esta combinación?

- App Router = layouts incorporados, metadatos, enrutamiento por localización
- Cloudflare = desplegado en el edge, costo cero, rápido

Sin backend, sin BD, sin costos de servidor. Solo entrega pura de utilidades.

## ✨ La IA me ayudó 10x: Así es cómo

> "La IA no te está reemplazando, está reemplazando las partes aburridas."

Usé **GitHub Copilot** dentro de VS Code para autocompletar:

- Entradas de formulario y manejadores de estado
- Funciones de utilidad (UUIDs, codificadores, etc.)
- Componentes de página boilerplate
- Plantillas de publicaciones de blog

Luego usé **Claude** (estilo agente) para:

- "¿Cómo comprimo una imagen del lado del cliente en el navegador?"
- "¿Cuál es la mejor manera de analizar cadenas cron en texto legible?"
- "¿Puedes traducir este archivo i18n al japonés y coreano?"

Incluso hice que Claude escribiera:

- FAQs
- Introducciones de publicaciones de blog
- Plantillas de etiquetas `meta` para 15 idiomas
- Esquemas JSON-LD para SEO

Copilot = generador de código
Claude = asistente basado en prompts
¿Juntos? Un código de trampa para desarrolladores en solitario.

## 🌍 Multilingüe: IA + Revisión manual

> "La traducción es fácil con IA. La traducción de calidad todavía requiere esfuerzo."

VictoryHub soporta **15 idiomas**:
🇬🇧 🇨🇳 🇯🇵 🇰🇷 🇩🇪 🇫🇷 🇪🇸 🇷🇺 🇳🇱 🇮🇹 🇵🇹 🇳🇴 🇸🇪 🇫🇮 🇩🇰

Mi flujo de trabajo:

1. Todo el texto en `en.json`
2. Usar Claude para traducir a cada localización objetivo
3. Revisar manualmente palabras sensibles al contexto
4. Verificación automática de claves faltantes mediante un script

El enrutamiento `i18n` del App Router + importaciones dinámicas hicieron que las rutas localizadas fueran limpias.

Bonus SEO: Agregué etiquetas `hreflang` + `alternate` para cada ruta.

## 🔍 SEO desde el día 1: Estructura antes que contenido

Si estás construyendo un sitio de herramientas y **no estás pensando en SEO**, estás desperdiciando su potencial.

Cosas que lancé temprano:

- ✅ `sitemap.xml` vía `next-sitemap`
- ✅ `robots.txt`
- ✅ Datos estructurados JSON-LD para herramientas + FAQ
- ✅ Títulos/descripciones meta por página en 15 idiomas (escritos por IA)
- ✅ Etiquetas canónicas + `hreflang`

> "¿Crees que el SEO es lento? Intenta hacerlo _después_ del lanzamiento."

### Consejo de valor:

Prompt para IA:

> "Escribe una meta descripción SEO para una página de \[herramienta\] en \[idioma\] en menos de 155 caracteres."

Funciona sorprendentemente bien. Todavía ajusto, pero es 10 veces más rápido.

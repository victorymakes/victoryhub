## ⚙️ Стек: Создан для скорости и SEO

- **Фреймворк:** Next.js 15 (App Router)
- **Хостинг:** Cloudflare Pages
- **CSS:** Tailwind
- **UI:** Shadcn
- **ИИ:** GitHub Copilot + Claude
- **CI/CD:** GitHub → Cloudflare адаптер (`opennextjs-cloudflare`)

Почему эта комбинация?

- App Router = встроенные макеты, метаданные, маршрутизация по локали
- Cloudflare = развертывание на edge, нулевая стоимость, быстрота

Без бэкенда, без БД, без затрат на сервер. Только чистая доставка утилит.

- "Как сжать изображение на стороне клиента в браузере?"

- "Можешь перевести этот i18n файл на японский и корейский?"

Я даже заставил Claude написать:

- FAQ
- Вступления к блог-постам
- Шаблоны `meta` тегов для 15 языков
- JSON-LD схемы для SEO

Copilot = генератор кода
Claude = ассистент на основе промптов

## 🔍 SEO с первого дня: Структура перед содержанием

Если вы создаете сайт с инструментами и **не думаете о SEO**, вы тратите его потенциал впустую.

Что я выпустил рано:

- ✅ `sitemap.xml` через `next-sitemap`
- ✅ `robots.txt`
- ✅ Структурированные данные JSON-LD для инструментов + FAQ
- ✅ Мета-заголовки/описания для каждой страницы на 15 языках (написанные ИИ)
- ✅ Канонические + `hreflang` теги

> "Думаете, SEO медленный? Попробуйте сделать его _после_ запуска."

### Ценный совет:

Промпт для ИИ:

> "Напиши SEO мета-описание для страницы \[инструмента\] на \[языке\] менее 155 символов."

Работает удивительно хорошо. Я всё ещё настраиваю, но это в 10 раз быстрее.

## 🤔 Всё ещё думаете, что карта сайта не имеет значения?

GoogleBot не может индексировать то, что не может найти.
Ваши инструменты = невидимы, если:

- Нет карты сайта
- Нет ссылок
- Нет внутренней маршрутизации

Исправление этого — 1 час работы. Не пропускайте это.

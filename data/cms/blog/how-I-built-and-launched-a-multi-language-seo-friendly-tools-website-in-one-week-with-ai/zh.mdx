## ⚙️ 技术栈：为速度和SEO而构建

- **框架：** Next.js 15（App Router）
- **托管：** Cloudflare Pages
- **CSS：** Tailwind
- **UI：** Shadcn
- **AI：** GitHub Copilot + Claude
- **CI/CD：** GitHub → Cloudflare 适配器（`opennextjs-cloudflare`）

为什么选择这个组合？

- App Router = 内置布局、元数据、按区域设置路由
- Cloudflare = 边缘部署、零成本、快速

没有后端，没有数据库，没有服务器成本。只有纯粹的工具提供。

## ✨ AI帮助我提高10倍效率：方法如下

> "AI不是在替代你，而是在替代枯燥的部分。"

我在VS Code中使用**GitHub Copilot**自动完成：

- 表单输入和状态处理器
- 实用函数（UUID、编码器等）
- 样板页面组件
- 博客文章模板

然后我使用**Claude**（代理风格）来：

- "如何在浏览器中客户端压缩图像？"
- "解析cron字符串为可读文本的最佳方式是什么？"
- "你能将这个i18n文件翻译成日语和韩语吗？"

我甚至让Claude编写：

- 常见问题解答
- 博客文章介绍
- 15种语言的`meta`标签模板
- 用于SEO的JSON-LD模式

Copilot = 代码生成器
Claude = 基于提示的助手
一起使用？独立开发者的秘密武器。

我的工作流程：

1. 所有文本放在`en.json`中
2. 使用Claude翻译成每个目标语言
3. 手动审核上下文敏感的词汇
4. 通过脚本自动检查缺失的键

App Router的`i18n`路由 + 动态导入使本地化路径变得简洁。

SEO加分：我为每个路由添加了`hreflang` + `alternate`标签。

## 🤔 还认为网站地图不重要？

- 没有内部路由

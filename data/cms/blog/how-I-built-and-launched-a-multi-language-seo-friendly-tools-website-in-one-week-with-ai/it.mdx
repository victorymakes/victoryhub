## ⚙️ Stack: Costruito per velocità e SEO

- **Framework:** Next.js 15 (App Router)
- **Hosting:** Cloudflare Pages
- **CSS:** Tailwind
- **UI:** Shadcn
- **IA:** GitHub Copilot + Claude
- **CI/CD:** GitHub → Adattatore Cloudflare (`opennextjs-cloudflare`)

Perché questa combinazione?

- App Router = layout incorporati, metadati, routing per località
- Cloudflare = distribuzione edge, costo zero, veloce

Nessun backend, nessun DB, nessun costo server. Solo pura distribuzione di utility.

## ✨ L'IA mi ha aiutato 10x: Ecco come

> "L'IA non sta sostituendo te, sta sostituendo le parti noiose."

Ho usato **GitHub Copilot** all'interno di VS Code per autocompletare:

- Input di form e gestori di stato
- Funzioni di utilità (UUID, encoder, ecc.)
- Componenti di pagina boilerplate
- Template di post del blog

Poi ho usato **Claude** (in stile agente) per:

- "Come comprimere un'immagine lato client nel browser?"
- "Qual è il modo migliore per analizzare stringhe cron in testo leggibile?"
- "Puoi tradurre questo file i18n in giapponese e coreano?"

Ho persino fatto scrivere a Claude:

- FAQ
- Introduzioni ai post del blog
- Template di tag `meta` per 15 lingue
- Schemi JSON-LD per SEO

Copilot = generatore di codice
Claude = assistente basato su prompt
Insieme? Un codice di imbroglio per sviluppatori solitari.

## 🔍 SEO dal giorno 1: Struttura prima del contenuto

Se stai costruendo un sito di strumenti e **non stai pensando al SEO**, stai sprecando il tuo potenziale.

Cose che ho lanciato presto:

- ✅ `sitemap.xml` tramite `next-sitemap`
- ✅ `robots.txt`
- ✅ Dati strutturati JSON-LD per strumenti + FAQ
- ✅ Titoli/descrizioni meta per pagina in 15 lingue (scritti dall'IA)
- ✅ Tag canonici + `hreflang`

> "Pensi che il SEO sia lento? Prova a farlo _dopo_ il lancio."

### Consiglio prezioso:

Prompt per l'IA:

> "Scrivi una meta descrizione SEO per una pagina di \[strumento\] in \[lingua\] con meno di 155 caratteri."

Funziona sorprendentemente bene. La modifico ancora, ma è 10 volte più veloce.

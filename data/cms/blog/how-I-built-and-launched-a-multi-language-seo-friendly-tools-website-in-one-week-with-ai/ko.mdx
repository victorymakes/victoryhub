## ⚙️ 스택: 속도와 SEO를 위해 구축

- **프레임워크:** Next.js 15 (App Router)
- **호스팅:** Cloudflare Pages
- **CSS:** Tailwind
- **UI:** Shadcn
- **AI:** GitHub Copilot + Claude
- **CI/CD:** GitHub → Cloudflare 어댑터 (`opennextjs-cloudflare`)

왜 이 조합인가요?

- App Router = 내장 레이아웃, 메타데이터, 로케일별 라우팅
- Cloudflare = 엣지 배포, 제로 비용, 빠름

백엔드 없음, DB 없음, 서버 비용 없음. 순수한 유틸리티 제공만 있습니다.

## ✨ AI가 10배 도움이 되었습니다: 방법은 다음과 같습니다

> "AI는 당신을 대체하는 것이 아니라, 지루한 부분을 대체하는 것입니다."

VS Code 내에서 **GitHub Copilot**을 사용하여 자동 완성했습니다:

- 폼 입력 및 상태 핸들러
- 유틸리티 함수 (UUID, 인코더 등)
- 보일러플레이트 페이지 컴포넌트
- 블로그 포스트 템플릿

그리고 **Claude**(에이전트 스타일)를 사용했습니다:

- "브라우저에서 클라이언트 측에서 이미지를 압축하는 방법은 무엇인가요?"
- "cron 문자열을 읽기 쉬운 텍스트로 파싱하는 가장 좋은 방법은 무엇인가요?"
- "이 i18n 파일을 일본어와 한국어로 번역해 주시겠어요?"

Claude가 작성한 것들:

- FAQ
- 블로그 포스트 인트로
- 15개 언어의 `meta` 태그 템플릿
- SEO를 위한 JSON-LD 스키마

Copilot = 코드 생성기
Claude = 프롬프트 기반 어시스턴트
함께? 솔로 개발자의 치트 코드입니다.

## 🌍 다국어: AI + 수동 검토

> "번역은 AI로 쉽습니다. 품질 좋은 번역은 여전히 노력이 필요합니다."

VictoryHub는 **15개 언어**를 지원합니다:
🇬🇧 🇨🇳 🇯🇵 🇰🇷 🇩🇪 🇫🇷 🇪🇸 🇷🇺 🇳🇱 🇮🇹 🇵🇹 🇳🇴 🇸🇪 🇫🇮 🇩🇰

제 워크플로우:

1. 모든 텍스트를 `en.json`에 넣기
2. Claude를 사용하여 각 대상 로케일로 번역
3. 컨텍스트에 민감한 단어를 수동으로 검토
4. 스크립트를 통해 누락된 키를 자동 확인

App Router의 `i18n` 라우팅 + 동적 임포트로 지역화된 경로가 깔끔해졌습니다.

SEO 보너스: 각 경로에 `hreflang` + `alternate` 태그를 추가했습니다.

## 🔍 첫날부터 SEO: 콘텐츠 전에 구조

도구 사이트를 구축하면서 **SEO에 대해 생각하지 않는다면**, 그 잠재력을 낭비하고 있는 것입니다.

초기에 출시한 것들:

- ✅ `next-sitemap`을 통한 `sitemap.xml`
- ✅ `robots.txt`
- ✅ 도구 + FAQ를 위한 JSON-LD 구조화 데이터
- ✅ 15개 언어로 된 페이지별 메타 제목/설명 (AI 작성)
- ✅ 정규 + `hreflang` 태그

> "SEO가 느리다고 생각하세요? 출시 _후에_ 해보세요."

### 가치 있는 팁:

AI를 위한 프롬프트:

> "\[언어\]로 155자 이내의 \[도구\] 페이지 SEO 메타 설명을 작성해주세요."

놀랍도록 잘 작동합니다. 여전히 조정하지만, 10배 빠릅니다.

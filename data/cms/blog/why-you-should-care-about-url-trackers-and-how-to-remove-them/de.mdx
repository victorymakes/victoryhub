# Warum URL-Tracker wichtig sind und wie man sie entfernt

Wenn Sie einen Link teilen oder anklicken, enthält er oft **Tracking-Parameter** im URL.
Diese _URL-Tracker_ (wie `utm_source` oder `fbclid`) werden von Unternehmen genutzt, um zu verfolgen, woher Sie kommen, was Sie anklicken und welche Kampagne Sie zu ihnen geführt hat.

Für Marketingzwecke nützlich, aber sie bergen erhebliche **Datenschutzrisiken**.

---

## 🔒 Wie URL-Tracker Ihre Privatsphäre beeinflussen

- **Offenlegung Ihres Surfverhaltens**: Tracker zeigen, von welcher Seite oder App Sie kommen.
- **Plattformübergreifende Profilbildung**: Werbetreibende verknüpfen Ihre Aktivitäten auf verschiedenen Websites.
- **Unübersichtliche URLs**: Lange, schwer lesbare Links voller Parameter.
- **Unerwünschte Datenweitergabe**: Informationen werden oft ohne Ihre Zustimmung an Dritte gesendet.

Kurz gesagt: URL-Tracker sind unsichtbare Brotkrumen, die Sie im Internet hinterlassen.

---

## 📑 Übersicht der wichtigsten URL-Tracking-Parameter

Diese Übersicht enthält die gängigsten Tracking-Parameter, deren Zweck und Verwendung. Sie basiert auf Standards, Analyseplattformen, Werbesystemen und SEO-Best Practices.

---

### 1. UTM-Parameter (Urchin Tracking Module)

Standard-Marketing-Parameter, unterstützt von Google Analytics und vielen anderen Plattformen.

| Parameter      | Beschreibung                                                    |
| -------------- | --------------------------------------------------------------- |
| `utm_source`   | Herkunft des Traffics (z.B. `google`, `newsletter`, `facebook`) |
| `utm_medium`   | Marketing-Medium (z.B. `email`, `cpc`, `social`)                |
| `utm_campaign` | Kampagnenname (z.B. `spring_sale`)                              |
| `utm_term`     | Suchbegriffe bei bezahlter Suche (z.B. `running+shoes`)         |
| `utm_content`  | Unterscheidung ähnlicher Inhalte/Links (z.B. `logolink`)        |

---

### 2. Werbe-Klick-IDs

Automatisch generierte Parameter von Werbeplattformen zur Klick-Attribution.

| Parameter | Plattform / Zweck           |
| --------- | --------------------------- |
| `gclid`   | Google Ads Klick-ID         |
| `dclid`   | Google Display Ads Klick-ID |
| `fbclid`  | Facebook/Instagram Klick-ID |
| `msclkid` | Microsoft/Bing Ads Klick-ID |
| `twclid`  | Twitter/X Klick-ID          |
| `yclid`   | Yandex Ads Klick-ID         |

---

### 3. Referral- & Affiliate-Tracking

Zur Zuordnung von Empfehlungen, Partnerschaften oder Affiliate-Kampagnen.

| Parameter              | Beschreibung               |
| ---------------------- | -------------------------- |
| `ref` / `referrer`     | Referral-Quelle            |
| `aff_id` / `affiliate` | Affiliate- oder Partner-ID |
| `partner`              | Partnerprogramm/-quelle    |
| `campaignid`           | Kampagnen-ID               |
| `adgroupid`            | Anzeigengruppen-ID         |
| `creative`             | Kreativ-Asset-ID           |

---

### 4. Experiment/A/B-Testing

Parameter zur Gruppenzuordnung bei Tests.

| Parameter              | Zweck                        |
| ---------------------- | ---------------------------- |
| `experiment` / `expid` | Name oder ID des Experiments |
| `variant`              | Variante oder Testversion    |
| `bucket` / `group`     | Testgruppe/Bucket-ID         |

---

### 5. Sitzungs- & Client-Tracking

Parameter zur Sitzungs- oder Client-Identifikation.

| Parameter           | Zweck                      |
| ------------------- | -------------------------- |
| `sessionid` / `sid` | Sitzungs-ID (heute selten) |
| `cid` / `clientid`  | Client- oder Nutzer-ID     |
| `tracking_id`       | Interne Tracking-ID        |

---

### 6. Plattform-spezifische Parameter

Spezielle Parameter von verschiedenen Plattformen oder Tools.

| Parameter          | Plattform / Zweck         |
| ------------------ | ------------------------- |
| `mc_eid`           | Mailchimp E-Mail-Tracking |
| `_hsenc` / `_hsmi` | HubSpot E-Mail-Tracking   |
| `igshid`           | Instagram Share-ID        |

---

### 7. Aktive vs. Passive Parameter

Ob sie die Seiteninhalte beeinflussen oder nur zum Tracking dienen:

- **Aktive Parameter**: Ändern Seiteninhalte oder Funktionen.
  Beispiele:
- `category=shoes`, `color=blue`, `sort=price_asc`, `page=2`, `lang=de`
- **Passive Parameter**: Dienen nur dem Tracking.
  Beispiele:
- `utm_*`, `gclid`, `ref`, `sessionid`

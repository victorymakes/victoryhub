# Почему важно заботиться о URL-трекерах и как их удалить

Когда вы делитесь или переходите по ссылке, она часто содержит **параметры отслеживания**.
Эти _URL-трекеры_ (например, `utm_source` или `fbclid`) позволяют компаниям отслеживать, откуда вы пришли, что вы нажимаете и какая реклама привела вас на сайт.

Для маркетинга это полезно, но для вас — это серьёзные **риски для приватности**.

---

## 🔒 Как URL-трекеры влияют на вашу приватность

- **Раскрытие ваших привычек**: Трекеры показывают, с какого сайта или приложения вы пришли.
- **Кросс-платформенное профилирование**: Рекламодатели связывают вашу активность на разных сайтах.
- **Загромождённые URL**: Длинные, трудночитаемые ссылки с множеством параметров.
- **Передача данных третьим лицам**: Информация часто уходит без вашего согласия.

Короче: URL-трекеры — это невидимые следы, которые вы оставляете в интернете.

---

## 📑 Справочник по параметрам URL-трекеров

Здесь собраны самые распространённые параметры отслеживания, их назначение и использование. Основано на стандартах, аналитических платформах, рекламных системах и лучших SEO-практиках.

---

### 1. UTM-параметры (Urchin Tracking Module)

Стандартные параметры для маркетинга, поддерживаются Google Analytics и другими платформами.

| Параметр       | Описание                                                        |
| -------------- | --------------------------------------------------------------- |
| `utm_source`   | Источник трафика (например, `google`, `newsletter`, `facebook`) |
| `utm_medium`   | Маркетинговый канал (например, `email`, `cpc`, `social`)        |
| `utm_campaign` | Название кампании или акции (например, `spring_sale`)           |
| `utm_term`     | Ключевые слова для платного поиска (например, `running+shoes`)  |
| `utm_content`  | Различие контента/ссылок для A/B теста (например, `logolink`)   |

---

### 2. Идентификаторы кликов рекламы

Автоматически генерируемые параметры для атрибуции кликов.

| Параметр  | Платформа / Назначение           |
| --------- | -------------------------------- |
| `gclid`   | Идентификатор клика Google Ads   |
| `dclid`   | Идентификатор Google Display Ads |
| `fbclid`  | Идентификатор Facebook/Instagram |
| `msclkid` | Идентификатор Microsoft/Bing Ads |
| `twclid`  | Идентификатор Twitter/X          |
| `yclid`   | Идентификатор Yandex Ads         |

---

### 3. Реферальные и партнёрские параметры

Для отслеживания рефералов, партнёров и аффилированных кампаний.

| Параметр               | Описание                            |
| ---------------------- | ----------------------------------- |
| `ref` / `referrer`     | Источник реферала                   |
| `aff_id` / `affiliate` | ID партнёра или канала              |
| `partner`              | Идентификатор партнёрской программы |
| `campaignid`           | ID рекламной кампании               |
| `adgroupid`            | ID группы объявлений                |
| `creative`             | ID креативного ресурса              |

---

### 4. Эксперименты / A/B тесты

Параметры для распределения пользователей по группам.

| Параметр               | Назначение                   |
| ---------------------- | ---------------------------- |
| `experiment` / `expid` | Название или ID эксперимента |
| `variant`              | Вариант или версия теста     |
| `bucket` / `group`     | ID группы эксперимента       |

---

### 5. Сессии и клиентские параметры

Для идентификации сессии или клиента (аналитика).

| Параметр            | Назначение                     |
| ------------------- | ------------------------------ |
| `sessionid` / `sid` | Идентификатор сессии           |
| `cid` / `clientid`  | ID клиента или пользователя    |
| `tracking_id`       | Внутренний ID для отслеживания |

---

### 6. Специфические параметры платформ

Особые параметры от разных сервисов и инструментов.

| Параметр           | Платформа / Назначение          |
| ------------------ | ------------------------------- |
| `mc_eid`           | Трекинг кликов Mailchimp        |
| `_hsenc` / `_hsmi` | Трекинг email HubSpot           |
| `igshid`           | Идентификатор шеринга Instagram |

---

### 7. Активные и пассивные параметры

По функционалу — влияют ли на контент или только для отслеживания:

- **Активные параметры**: Меняют содержимое или функционал страницы.
  Примеры:
- `category=shoes`, `color=blue`, `sort=price_asc`, `page=2`, `lang=de`
- **Пассивные параметры**: Только для отслеживания, не меняют контент.
  Примеры:
- `utm_*`, `gclid`, `ref`, `sessionid`

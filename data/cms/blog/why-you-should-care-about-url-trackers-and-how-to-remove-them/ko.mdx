# URL 추적기를 신경 써야 하는 이유와 제거 방법

링크를 공유하거나 클릭할 때, URL에는 **추적 파라미터**가 숨어 있는 경우가 많습니다.
이러한 _URL 추적기_(`utm_source`, `fbclid` 등)는 기업이 당신의 유입 경로, 클릭 내역, 어떤 광고나 캠페인에서 왔는지 추적하는 데 사용됩니다.

마케터에게는 유용하지만, **개인정보 보호 측면에서는 주의가 필요**합니다.

---

## 🔒 URL 추적기가 개인정보에 미치는 영향

- **브라우징 습관 노출**: 어떤 사이트나 앱에서 왔는지 드러남
- **크로스 플랫폼 프로파일링**: 광고주가 여러 사이트의 활동을 연결
- **지저분하고 긴 URL**: 불필요한 파라미터로 링크가 복잡해짐
- **원치 않는 데이터 공유**: 동의 없이 제3자에게 정보가 전달됨

즉, URL 추적기는 인터넷 곳곳에 남기는 보이지 않는 흔적입니다.

---

## 📑 주요 URL 추적 파라미터 참고

인터넷에서 자주 쓰이는 URL 추적 파라미터와 그 용도를 정리했습니다. 표준 용어, 분석 플랫폼, 광고 시스템, SEO 베스트 프랙티스를 참고했습니다.

---

### 1. UTM 파라미터 (Urchin Tracking Module)

Google Analytics 등에서 지원하는 표준 마케팅 추적 파라미터.

| 파라미터       | 설명                                              |
| -------------- | ------------------------------------------------- |
| `utm_source`   | 유입 경로(예: `google`, `newsletter`, `facebook`) |
| `utm_medium`   | 마케팅 매체(예: `email`, `cpc`, `social`)         |
| `utm_campaign` | 캠페인/프로모션 식별자(예: `spring_sale`)         |
| `utm_term`     | 유료 검색 키워드(예: `running+shoes`)             |
| `utm_content`  | A/B 테스트용 콘텐츠 구분(예: `logolink`)          |

---

### 2. 광고 클릭 식별자

광고 플랫폼이 자동 생성하는 클릭 추적 파라미터.

| 파라미터  | 플랫폼/용도                    |
| --------- | ------------------------------ |
| `gclid`   | Google Ads 클릭 식별자         |
| `dclid`   | Google 디스플레이 광고 식별자  |
| `fbclid`  | Facebook/Instagram 클릭 식별자 |
| `msclkid` | Microsoft/Bing 광고 식별자     |
| `twclid`  | Twitter/X 클릭 식별자          |
| `yclid`   | Yandex 광고 식별자             |

---

### 3. 추천 및 제휴 추적

추천, 파트너십, 제휴 캠페인에 사용.

| 파라미터               | 설명                        |
| ---------------------- | --------------------------- |
| `ref` / `referrer`     | 추천 소스 식별자            |
| `aff_id` / `affiliate` | 제휴 채널/파트너 ID         |
| `partner`              | 파트너 프로그램/소스 식별자 |
| `campaignid`           | 광고 캠페인 ID              |
| `adgroupid`            | 캠페인 내 그룹 ID           |
| `creative`             | 크리에이티브 자산 ID        |

---

### 4. 실험/A/B 테스트

실험 그룹 할당용 파라미터.

| 파라미터               | 용도                  |
| ---------------------- | --------------------- |
| `experiment` / `expid` | 실험명 또는 ID        |
| `variant`              | 변형 또는 테스트 버전 |
| `bucket` / `group`     | 실험 그룹 ID          |

---

### 5. 세션 및 클라이언트 추적

세션 또는 클라이언트 식별용(분석 목적).

| 파라미터            | 용도                     |
| ------------------- | ------------------------ |
| `sessionid` / `sid` | 세션 식별자(현재는 드묾) |
| `cid` / `clientid`  | 클라이언트/사용자 ID     |
| `tracking_id`       | 내부 추적용 식별자       |

---

### 6. 플랫폼별 특수 식별자

특정 플랫폼/도구의 특수 파라미터.

| 파라미터           | 플랫폼/용도                  |
| ------------------ | ---------------------------- |
| `mc_eid`           | Mailchimp 이메일 클릭 추적   |
| `_hsenc` / `_hsmi` | HubSpot 이메일 추적 파라미터 |
| `igshid`           | Instagram 공유 식별자        |

---

### 7. 활성 vs. 수동 파라미터

기능 기준—콘텐츠에 영향 주는지, 단순 추적인지:

- **활성 파라미터**: 페이지 내용/기능을 변경함
  예시:
- `category=shoes`, `color=blue`, `sort=price_asc`, `page=2`, `lang=de`
- **수동 파라미터**: 내용 변경 없음, 추적만 함
  예시:
- `utm_*`, `gclid`, `ref`, `sessionid`
